# Hawkeye mobile (Capacitor shell)

Wraps the existing PWA in `../app` into native iOS + Android **without a rewrite**
(see `docs/MOBILE-APP-ARCHITECTURE.md`). The web bundle is the source of truth;
this project only adds the native shell + plugins.

## What's committed vs generated

Committed: `package.json`, `capacitor.config.json`, this README.
Generated by Capacitor (gitignored — recreate anytime): `node_modules/`,
`android/`, `ios/`.

## First-time setup

```bash
cd mobile
npm install
npx cap add android          # creates android/ (needs Android Studio to BUILD)
npx cap add ios              # on macOS only (needs Xcode + CocoaPods)
npx cap sync                 # copies ../app into the native projects
```

## Build / run

- **Android:** `npx cap open android` → run/build in Android Studio (needs the
  Android SDK + a device/emulator). CLI: `npx cap run android`.
- **iOS:** `npx cap open ios` on a Mac → run in Xcode. Needs an Apple Developer
  account ($99/yr) to sign and ship.

Rebuild after any change to `../app`: `npx cap sync`.

## How the shell talks to the backend

The bundle is served locally in the shell (offline-capable), so there is **no
same-origin server**. `app/native.js` (a strict no-op on the web) rewrites
leading-slash URLs to `https://hawkeye.com.ng` when running natively, and
`CapacitorHttp` (enabled in `capacitor.config.json`) routes fetch through native
HTTP so it's CORS-safe; requests still traverse Cloudflare (origin lock intact).

## Native features — status

Scaffold wires the plugins (camera, geolocation, preferences, network, push).
The capability seams in `app/native.js` (`window.HAWKEYE.capabilities`) are the
integration points for the next phase, per architecture §3:

- [ ] Camera → native capture-only plugin (**must keep** compress → hash → sign →
      upload order; the ledger content-addresses the compressed bytes).
- [ ] Signing key → iOS Keychain / Android Keystore (Secure Enclave / TEE).
- [ ] Push → FCM (Android) / APNs (iOS); backend `device_push_tokens` + send helper.
- [ ] Offline outbox → queue signed reports, flush on reconnect (idempotent on
      `entry_hash`).
